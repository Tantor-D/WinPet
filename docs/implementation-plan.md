# WinPet 实施计划

## 1. 产品目标

WinPet 自动判断用户是否正在使用电脑，跟踪连续工作与有效休息，在久坐时通过
桌宠和系统通知提醒用户，并提供日、周、月维度的本地统计。

首发平台为 Windows。macOS 不在首个版本的交付范围内，但平台能力必须从第一天
通过接口隔离，避免 UI、计时和统计代码依赖 Win32。

## 2. 统计口径

- **在机时长**：系统处于解锁且未休眠状态的时间。
- **活跃时长**：最近一段时间内存在键鼠输入的时间。
- **短暂空闲**：没有输入，但尚未达到有效休息阈值。
- **连续工作时长**：两次有效休息之间的在机使用时长。
- **有效休息**：无输入、锁屏或休眠持续达到用户设置的休息阈值。
- **超时工作**：连续工作超过最长工作时长后仍在使用电脑的时间。
- **高活跃时段**：按小时聚合后，平均活跃分钟较高的时段。

这些指标描述电脑使用行为，不宣称能够判断用户是否真正专注或从事工作。

## 3. 默认规则

- 活动判定窗口：60 秒。
- 最长连续工作：45 分钟。
- 提前警告：5 分钟。
- 有效休息：5 分钟。
- 活动采样：5 秒。
- 数据汇总粒度：1 分钟。

所有阈值均由用户配置。锁屏和休眠计入休息，不计入工作。

## 4. 系统结构

```text
平台活动采样
    ↓
WorkSessionEngine
    ├── 状态变化 → PetStateEngine → 桌宠动画
    ├── 提醒事件 → ReminderService → 通知/声音
    └── 时间片段 → ActivityAggregator → SQLite
                                      ↓
                              StatisticsService
                                      ↓
                       今日 / 时间轴 / 周月趋势
```

### Core

- `IActivityMonitor`：输出统一的活动快照。
- `WorkSessionEngine`：判断工作、警告、超时、休息和重置。
- `PetStateEngine`：把业务状态映射成可配置的桌宠状态。
- `StatisticsService`：计算每日摘要、趋势、热力图和健康指标。

### Platform

- Windows：`GetLastInputInfo`、会话锁定事件、电源事件。
- macOS：系统 idle time、workspace session 与 sleep/wake 事件。

### Infrastructure

- SQLite 原始数据和可重算汇总。
- JSON 用户设置及主题清单。
- CSV/JSON 数据导出。

### Desktop

- 透明桌宠窗口。
- 系统托盘快捷操作。
- 设置窗口。
- 今日概览、单日时间轴、周/月趋势和小时热力图。

## 5. 数据模型

### activity_buckets

每分钟记录活跃、短暂空闲、锁屏、休眠和超时秒数，并关联工作轮次。

### work_sessions

记录每轮连续工作的起止时间、活跃时长、超时时长、提醒次数和结束原因。

### rest_sessions

记录休息起止时间、时长及是否达到自动重置标准。

### reminder_events

记录提醒类型、用户动作和从提醒到离开电脑的响应时间。

### daily_summaries

作为可重新生成的查询缓存，提升统计页加载速度，不作为唯一事实来源。

数据库时间统一保存为 UTC；展示和按日聚合时使用用户时区。

## 6. 交付阶段

### M1：核心规则

- 建立解决方案和项目边界。
- 完成活动快照、设置和会话状态模型。
- 实现工作/休息状态机及单元测试。
- 定义平台和持久化接口。

验收：关键规则测试通过，核心项目不引用 Avalonia、SQLite 或 Win32。

### M2：Windows 活动检测

- 实现最后输入时间查询。
- 处理锁屏、解锁、休眠和唤醒。
- 建立后台采样服务。

验收：Windows 上可持续输出正确活动快照，不采集具体输入内容。

### M3：本地数据

- 建立 SQLite schema 和迁移。
- 保存分钟桶、工作轮次、休息与提醒。
- 支持异常退出后的会话恢复。

验收：重启后数据不丢失，日报可从原始数据重新生成。

### M4：桌宠与提醒

- 透明、无边框、可拖动、可选置顶的桌宠窗口。
- 工作、空闲、警告、需要休息、休息中和欢迎回来状态。
- 托盘、手动重置、暂停、延后和退出。
- JSON 主题包，支持 PNG、GIF 或序列帧。

验收：状态变化与计时状态一致，主题素材无需改代码即可替换。

### M5：统计视图

- 今日概览卡片。
- 单日时间轴。
- 周/月趋势。
- 小时活跃热力图。
- 久坐率、休息达标率、提醒响应时间。

验收：统计结果与原始分钟桶抽查一致，指标口径可在界面查看。

### M6：Windows 发布

- 设置迁移、开机启动和单实例运行。
- 数据导出、保留期和清除功能。
- 打包、安装及端到端验证。

### M7：macOS 移植

- 实现 macOS 平台接口。
- 适配菜单栏、通知、开机启动和窗口层级。
- 校准 Windows 与 macOS 的活动判定一致性。

## 7. 首批风险

- 系统睡眠、时间跳变和时区变化可能导致错误计时，必须使用单调时间或限制采样跨度。
- “电脑活跃”不等于“生产力”，所有文案必须避免过度推断。
- 透明窗口和点击穿透在 Windows/macOS 上行为不同，需要平台级验证。
- 分钟桶应保留可重算基础数据，不能只保存日报结果。
